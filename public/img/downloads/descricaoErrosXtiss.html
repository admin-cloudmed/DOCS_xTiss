Erros de validação de arquivos no xTiss e suas respectivas descrições.
E1 - Selecione um arquivo xml compactado para enviar.
Descrição: O prestador deve selecionar um arquivo compactado (extensão .zip) antes de clicar no
botão ‘Validar Arquivo’ para enviar o arquivo.
E2 - O arquivo deve vir com a extensão zip.
Descrição: O arquivo selecionado pelo prestador para o envio deve estar na extensão “.zip”
(compactado).
E4 - O arquivo compactado não pode conter mais de um arquivo xml.
Descrição: Dentro do arquivo compactado (extensão .zip) selecionado pelo Prestador, deve haver
apenas UM arquivo xml.
V1 - Arquivo xml não possui a tag "ans:destino".
Descrição: O arquivo xml deve conter a tag ”ans:destino” para a validação do código da ANS do
arquivo.
V3/V8/V16 - Arquivo xml não possui a tag "ans:mensagemTISS".
Descrição: O arquivo xml deve conter a tag “ans:mensagemTISS” para a validação do hash e
schema.
V4 - Hash do arquivo não confere com o hash original.
Descrição: Hash gerado pelo xTiss não confere com o hash apresentado na tag “ans:hash” do
arquivo. O arquivo pode ter sido alterado após sua produção, ou, o padrão utilizado para gerar o
hash no software do prestador está incorreto (o padrão de validação de Hash utilizado pelo xTiss é o
MD5 - ISO-8859-1).
V5 - Este arquivo já foi enviado anteriormente.
Descrição: O arquivo que o prestador está tentando enviar já foi postado anteriormente. Um
arquivo validado com sucesso só pode ser enviado uma vez no xTiss.
V6 - Versão do Schema ANS (nº versão) não cadastrado no sistema.
Descrição: A versão do schema TISS do arquivo xml não está cadastrado no xTiss. Entre em
contato com o administrador do sistema para que o cadastro seja efetuado.
V7 - A versão do schema não é válida, pois não se encontra dentro do período máximo de
usabilidade.
Descrição: A versão do schema TISS do arquivo xml não é valida, pois de acordo com as
configurações da singular o seu prazo de validade está ultrapassado. Os arquivos devem ser
enviados em uma versão mais recente.
V9 - Este arquivo não pertence ao hospital cujo seu login está relacionado.
Descrição: O arquivo que o prestador está tentando enviar pode não ser do seu hospital/clínica de
origem. Possíveis problemas: - Relacionamento entre login e registro do prestador incorreto; -
Código do prestador na operadora do arquivo xml não existe no xTiss ou está diferente do cadastro
existente; - CNPJ do prestador do arquivo xml não existe no xTiss ou está diferente do cadastro
existente.
V10 - Arquivo xml não possui a tag "ans:origem".
Descrição: O arquivo xml deve conter a tag ”ans:origem” para a validação de prestador enviando
arquivo de seu hospital/clínica.
V11 - O código da ANS (código) não está cadastrado no sistema.
Descrição: O código da ANS apresentado na tag “ans:registroANS” do arquivo não está cadastrado
no xTiss. Possíveis problemas: código ANS do arquivo xml está incorreto, ou, código não existente
no xTiss: entre em contato com o administrador do sistema para que o cadastro seja efetuado.
V12 - Arquivo xml não possui a tag "ans:registroANS".
Descrição: O arquivo xml deve conter a tag “ans:registroANS” para a validação do código da ANS
do arquivo.
V13/V31/V37 - Arquivo xml não possui a tag "ans:guiaFaturamento".
Descrição: O arquivo xml deve conter a tag “ans:guiaFaturamento” para a validação do código da
ANS do arquivo e dos números de guias/senhas do arquivo.
V14 - Configuração de Tipos de Guias não cadastrada para essa singular.
Descrição: O cadastro da configuração de “Tipos de Guias” é obrigatório para a singular. Entre em
contato com o administrador do sistema para que o cadastro seja efetuado.
V15 - Arquivo XML não está na estrutura correta. Detalhes: (erro).
Descrição: Arquivo xml não se encontra na estrutura correta para a validação de schema TISS.
V17 - Arquivo com dados inválidos em (tag do xml), na linha: (nº linha).
Descrição: Erro na validação com o schema da TISS: na linha “x” existe um dado fora do padrão
esperado pelo schema TISS. Veja na especificação do esquema o tipo de dado que deve ser enviado
na tag em questão.
V18 - O Arquivo xml contém mais de um tipo de conta diferente.
Descrição: O arquivo xml não pode conter mais de um tipo de conta (sp/sadt, internação, honorário,
consulta) diferente dentro do mesmo arquivo.
V19 - O arquivo XML contém uma quantidade de contas maior que a permitida. Número de contas
encontrado: (nº). Número máximo de contas permitido: (nº configurado).
Descrição: O número de contas do arquivo xml é superior ao número máximo de contas que foi
configurado pela singular no xTiss.
V20 - O Arquivo XML não contém nenhum tipo de conta.
Descrição: O arquivo xml deve conter ao menos uma conta para ser validado.
V21 - O número de CPF (nº) não é válido.
Descrição: O número do CPF “x” não é válido de acordo com os padrões de verificação nacional.
V22 - Arquivo xml não possui a tag "ans:cpf".
Descrição: O arquivo xml deve conter a tag “ans:cpf” para a validação dos números de CPF do
arquivo.
V23 - O número de CNPJ (nº) não é válido.
Descrição: O número do CNPJ “x” não é válido de acordo com os padrões de verificação nacional.
V24 - Arquivo xml não possui a tag "ans:CNPJ".
Descrição: O arquivo xml deve conter a tag “ans:CNPJ” para a validação dos números de CNPJ do

V25 - O Código do CID (código) não é válido.
Descrição: O código do CID “x” não é válido, pois não se encontra na tabela de CID-10.
V26 - Arquivo xml não possui a tag "ans:codigoDiagnostico".
Descrição: O arquivo xml deve conter a tag “ans:codigoDiagnostico” para a validação dos códigos
